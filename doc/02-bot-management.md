# Bot 管理

## Bot 列表

进入 **Bots** 页面后，你会看到所有 Bot 以卡片网格的形式排列。

每张卡片包含以下信息：
- **头像和名称**
- **类型标签**：蓝色标签表示 `personal`（私人），绿色标签表示 `public`（公开）。
- **AI 模型标签**：紫色标签显示 Bot 使用的对话模型名称（如 `gpt-4o`）。
- **状态**：创建中、活跃、停用等。

### 搜索

页面顶部的搜索框支持按以下字段模糊搜索：
- Bot 名称
- Bot ID
- Bot 类型

### 自动刷新

如果有 Bot 处于 `creating`（创建中）或 `deleting`（删除中）状态，列表会每 2 秒自动刷新一次，直到操作完成。

## 创建 Bot

点击右上角 **创建 Bot** 按钮，分两步完成：

### 第一步：选择模板

- 顶部标签可按分类筛选（如 all、productivity、assistant、creative 等）。
- 模板来自服务器 API（`GET /templates`），展示名称、图标和简介。
- 你也可以选择 **空白 Bot**，从零开始配置。

### 第二步：填写信息

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 显示名称 | Bot 的名字，用于列表和对话界面展示 | 是 |
| 头像 URL | 一个图片链接，作为 Bot 的头像 | 否 |
| 类型 | `personal`（仅创建者可用）或 `public`（所有用户可访问） | 是 |
| 特权模式 | 开启后 Bot 拥有更高的系统权限 | 否 |

选择模板后，Bot 会自动应用模板中预设的系统提示词、工具和配置。

## Bot 详情页

点击任意 Bot 卡片进入详情页，页面顶部显示：
- Bot 头像（悬停后可点击编辑）
- 显示名称（点击编辑按钮可内联修改）
- 状态徽标
- 类型徽标

### 标签页

详情页包含以下标签（Tab）：

| 标签 | 说明 |
|------|------|
| 概览 | 运行状态检查、进化摘要、心跳摘要、容器能力 |
| 进化 | 自进化机制的状态、实验记录、历史 |
| 记忆 | 浏览、编辑、压缩记忆，向量可视化 |
| 渠道 | 管理 Telegram、飞书等平台接入 |
| 文件 | 浏览和编辑容器内的人格文件 |
| 容器 | 容器详细信息、启停、快照管理 |
| MCP | 内置工具列表和外部 MCP 服务器配置 |
| 技能 | 自定义技能的管理 |
| 子智能体 | 管理 Bot 的子智能体 |
| 心跳 | 心跳触发器配置 |
| 历史 | 查看所有对话记录 |
| 设置 | 模型、行为参数、功能开关、删除 Bot |

### 概览标签

概览页提供以下信息：

**运行检查**：自动检测 Bot 的健康状态，包括：
- 容器是否初始化
- 容器记录是否正常
- 容器任务状态
- 数据路径是否可用
- Bot 是否处于删除状态

每个检查项显示 OK / Warning / Error 状态。

**容器能力**：展示 Bot 支持的功能模块：
- agent-browser（网页浏览）
- Actionbook（操作手册）
- smart-web-fetch（智能网页抓取）
- ClawHub（数据抓取）
- OpenViking（高级功能）

## 删除 Bot

在 Bot 详情页 → **设置** 标签最下方的 **危险区域**，点击删除按钮并确认即可删除 Bot。删除操作会同时清理关联的容器和数据。
