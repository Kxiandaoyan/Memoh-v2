# Bot 设置详解

进入 Bot 详情页 → **设置** 标签，可以配置 Bot 的所有参数。设置分为四个区域。

## 模型配置

| 字段 | 说明 |
|------|------|
| 对话模型 (Chat Model) | Bot 对话时使用的主要 AI 模型，如 `gpt-4o`、`claude-3.5-sonnet` |
| 记忆模型 (Memory Model) | 用于记忆提取和整理的模型 |
| 嵌入模型 (Embedding Model) | 用于将文本转换为向量的模型（记忆语义搜索依赖此模型） |
| VLM 模型 | 视觉语言模型，用于处理图片相关的任务 |
| 搜索提供方 (Search Provider) | Bot 使用的搜索引擎，如 Brave Search |

所有模型下拉框的选项来自 **模型管理** 页面中已添加的模型。如果下拉框为空，请先到模型管理页面添加对应类型的模型。

## 行为配置

| 字段 | 说明 |
|------|------|
| 语言 (Language) | Bot 的默认回复语言 |
| 任务提示词 (Task) | Bot 的系统提示词，定义其行为和角色。这是 Bot 人格的核心 |
| 最大上下文加载时间 (Max Context Load Time) | 加载对话上下文的超时时间（毫秒），设为 0 表示不限制 |

## 功能开关

以下功能均通过开关（Switch）控制：

### 允许自进化 (Allow Self Evolution)
- 默认：**开启**
- 开启后 Bot 可以通过心跳触发自进化流程，自动调整自身的身份文件和实验记录。
- 详情参考 [自进化与心跳](09-evolution-heartbeat.md)。

### 启用 OpenViking
- 默认：**关闭**
- 开启后 Bot 可以使用 OpenViking 功能模块。

### 特权模式 (Privileged)
- 默认：**关闭**
- 开启后 Bot 拥有更高的系统权限，可以执行更多操作。

### 允许访客 (Allow Guest)
- 默认：**关闭**
- 仅在 Bot 类型为 `public` 时显示。
- 开启后，未注册的用户也可以通过平台渠道（如 Telegram）与此 Bot 对话。

### 群聊需要 @提及 (Group Require Mention)
- 默认：**开启**
- 控制 Bot 在群聊中的响应行为：
  - **开启**（默认）：Bot 在群聊中只回复以下情况：
    - 被 @提及
    - 被引用回复
    - 收到命令前缀消息
  - **关闭**：Bot 会回复群聊中所有人类用户的消息（但不会回复其他 Bot 的消息，以防止无限循环）。
- 即使关闭了此选项，@提及和引用回复仍然始终有效。
- 详情参考 [平台接入 - 群聊行为](05-channel-integration.md#群聊行为)。

## 危险区域

页面底部的 **删除 Bot** 按钮，点击后会弹出确认对话框。确认后将永久删除 Bot 及其所有数据（容器、记忆、对话记录等）。此操作不可撤销。
