# Token 用量

## 进入 Token 用量页面

点击左侧导航的 **Token 用量** 进入。

## 概览卡片

页面顶部显示三个汇总卡片：

| 卡片 | 说明 |
|------|------|
| 总 Token | 所选时间范围内消耗的 Token 总数 |
| 输入 Token (Prompt) | 发送给模型的 Token 数量 |
| 输出 Token (Completion) | 模型生成的 Token 数量 |

## 时间范围

右上角可选择统计周期：
- **7 天**
- **30 天**
- **90 天**

切换后所有图表和数据自动刷新。

## 每日趋势图

折线图展示所选时间范围内每天的 Token 消耗趋势。

### 多 Bot 对比

如果有多个 Bot，图表会为每个 Bot 绘制独立的折线，方便对比不同 Bot 的用量差异。

### 三种线型

每个 Bot 使用三种线型区分：
- **实线**：总 Token
- **虚线**：输入 Token
- **点线**：输出 Token

### 悬停提示

鼠标悬停在数据点上会显示该天的详细数值。

## 模型分布

### 饼图

圆形百分比图展示各模型的 Token 占比，直观呈现哪个模型消耗最多。

### 详细表格

饼图下方的表格列出每个模型的：
- 输入 Token
- 输出 Token
- 总 Token
- 占比百分比

## Token 的计算

Token 用量在每次 Bot 回复时自动记录：
- 输入 Token：包括系统提示词、对话历史、记忆内容、用户消息等发送给模型的全部内容。
- 输出 Token：模型生成的回复内容。
- 数据来源于 AI 模型 API 返回的 `usage` 字段，确保与实际计费一致。
